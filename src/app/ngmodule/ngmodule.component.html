NgModule

Modules in Angular serve two purposes, first is to define which resources are available for a set of components, and secondly modules also allow for chunks of code to be defined for reuse by other Angular applications. All Angular projects must have at least one module, and the base module must select a component to be bootstrapped. Angular will then render the application starting with the identified component.

Another common use for modules is to create routing modules. The purpose of this is to define routing for when a module is lazy loaded. Inside a Routing Module you will generally define child routes, which will change which components are rendered based on URL changes. These routing modules are then imported into modules which define component dependencies. This allows for separation of concerns when defining reusable chunks of code and the routing logic.
