Services

Services in Angular are responsible for data, this includes application state and data from external APIs. Services are declared with the @Injectable decorator are added to an Angular application by including the service in the list providers inside of a module. Services are made available to other parts of the application through Angular's dependency injection system. This will ensure that if a service is pre-existing that a new one won't be recreated, making services work as singletons that can be shared. Services can also be injected Components, Directives, and other services. Services can also easily be mocked inside of a module by specifying that a different service should be used when the existing one is called, this is great for testing.

There is one caveat in terms of ensuring that services act as singletons throughout your application when using lazy loading. Each time a module is lazy loaded it creates a new lexical scope and any services provided by the lazily loaded module will be recreated. The proper work-around for this is to not include providers in lazily loaded modules unless a static forRoot function is called. When a component that is lazily loaded tries to call a service that doesn't exist in the current scope it will look up the tree to parent scopes to see if the service exists. It's best practice to only include services as privders at the root module of your application.

Services in Angular often use RxJS Observables to store and transmit data, see the Advanced section for details on RxJS.
