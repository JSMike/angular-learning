Testing

Angular was designed to be easily testable. The dependency injection system allows for services to be easily mocked and the Angular Testbed system allows for components to easily be created in isolation. There are two basic forms of testing a web application, unit testing, and scenario (end-to-end) testing. These forms of testing require different toolsets. Most developers focus on unit-testing all of the functions inside of their application while QA engineers focus on setting up scenarios to ensure workflows can be completed.

Common tools for unit testing include Karma, which loads the tests into a browser, Mocha, which is a framework for writing test cases, Chai, an assertion library, Sinon, which is used for creating mocks of objects, and Jasmine, which is a test framework which includes its own assertion and mocking functions. Depending on how you started out your project some of these libraries may have been included for you. For example, the Angular CLI includes Karma and Jasmine for running tests.

Angular has a popular tool for end-to-end testing called Protractor. Protractor uses a webdriver to load your Angular application into a browser and then run through a list of instructions to select elements on the page and fires off events such as clicking, typing, or moving the mouse. The tests then check if the page displays the expected data. For example, filling out a form and expecting either errors or a successful submission.
